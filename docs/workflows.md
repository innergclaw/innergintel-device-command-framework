# Workflows

## Client Website Build

1. Capture request on iPhone or Telegram.
2. Build on MacBook 1 with Codex.
3. Push to GitHub.
4. Review on MacBook 2.
5. Check mobile on iPhone.
6. Send client update through Telegram or direct message.

## Domain + GitHub Pages Deploy

1. Add/update repo files on MacBook 1.
2. Push to GitHub.
3. Configure DNS with human confirmation.
4. Verify from MacBook 2.
5. Check mobile from iPhone.
6. Log final domain and repo in InnerG Intel.

## Codex Skill Product Build

1. Define target user and pain point.
2. Build skill repo on MacBook 1.
3. Add install script and README.
4. Test sample data locally.
5. Push to GitHub.
6. Install into local Codex skills.
7. Add product to InnerG Intel catalog.

## Finance / Holdings Report

1. Refresh holdings from Codex.
2. Compare to latest log.
3. Send report to Telegram.
4. Flag big changes or manual action needed.
5. Keep approvals human-confirmed.

## Mobile Review Loop

1. Open live URL on iPhone.
2. Capture screenshot or screen recording.
3. Send to Codex.
4. Fix on MacBook 1.
5. Push.
6. Re-check on iPhone.
